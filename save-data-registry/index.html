    Як записати в реєстр?

    Передумови

    Для запису даних процесу в реєстр необхідно додати на схему елемент Подія (Event) з типом Запит.

    Опис елементів процесу та роботи з редактором див. на сторінці Як налаштувати процес?

    Запис в реєстр

    Для налаштування запису в реєстр виконайте такі дії:

    1) В схемі процесу оберіть елемент Подія (Event) із типом Запит.

    2) У панелі налаштування, що відкрилась праворуч, перейдіть на вкладку Схема події.

    3) Відкривається вікно для введення коду у форматі JSON.

    4) Додайте код зі спеціальним синтаксисом. Перелік параметрів, що підтримуються.

    5) Збережіть написаний код, натиснувши на піктограму Зберегти.

    Параметри, які підтримуються

    1) saveToRegisters — обовʼязковий параметр, який приймає рядок функції або обʼєкт з даними мапінгу:

    • варіант: рядок функції — описується рядкова функція JavaScript, яка приймає один аргумент — documents (масив документів процесу). Результатом опрацювання буде об'єкт даних для запису в реєстр, або об'єкт з даними мапінгу на схему ключа реєстру. Результат запису функції містить такі параметри:

      • registerId — обовʼязковий параметр для визначення, в який реєстр буде відбуватися запис даних за його ідентифікатором. Тип — number;
      • keyId — обовʼязковий параметр для визначення, в який ключ реєстру буде відбуватися запис даних за його ідентифікатором. Тип — number;
      • allowTokens — необовʼязковий параметр, який приймає перелік токенів підрозділів для надання доступу до цього запису в реєстрі. Тип — array;

      Зверніть увагу!

      Якщо в підрозділа є дозвіл на доступ до реєстру, але його немає в цьому переліку, то записи можуть бути недоступними.

      Якщо не встановлювати цей параметр, то записи будуть доступними для всіх підрозділів, які мають доступ в налаштуваннях ключа реєстру.

      • data — обовʼязковий параметр, який є об'єктом із переліком даних полів для запису їх в реєстр. Тип - object.

        "saveToRegisters": "
            (documents) => {
                const document = documents.find((item => item.documentTemplateId === 100));
                const country = document?.data?.countryInfo;
                const {
                    name,
                    code,
                    timeZone
                } = country;
                return {
                    registerId: 123,
                    keyId: 1231,
                    allowTokens: ['token1', 'token2', 'tokenN'],
                    data: {
                        name,
                        code: String(code),
                        timeZone
                    }
                }
            };
        "
    }

    • варіант: обʼєкт з даними мапінгу — результат запису функції містить такі параметри:

      • параметри registerId, keyId, allowTokens — аналогічні до варіанту, коли параметр приймає рядок функції;

      Info

      Параметр allowTokens також може приймати рядок функції, що приймає два аргументи: documents (масив документів поточного процесу) та events (масив подій поточного процесу) результатом якої має бути масив токенів підрозділу;

      • person — необовʼязковий параметр, який має тип object або є рядком функції, що приймає два аргументи documents (масив документів поточного процесу) та events (масив подій поточного процесу) з даними користувача, відповідального за створення запису. Формат запису: {id, name}, де: id — ідентифікатор користувача, що здійснив запис, name — ПІБ користувача, що здійснив запис. Результатом виконання має бути обʼєкт із заповненими параметрами {id, name} відповідно. Параметр необхідний у випадках, коли потрібно фіксувати, хто саме зробив запис до реєстру. Фіксується в історії змін, оскільки за замовчуванням це system.

      • map — обовʼязковий параметр, який є об'єктом мапінгу, параметри якого однойменні з переліком в схемі ключа реєстру. Тип — object. Значення мають такі варіанти запису:

        • рядок зі шляхом до необхідного значення документа по процесу — documents.<document-template-id>.<inner-document-path>, де:

          • documents — параметр зі значенням масиву документів процесу;

          • document-template-id — ідентифікатор номера процесу documentTemplateId, з якого потрібно отримати дані. Значення можна знайти в Логах процесу;

          • inner-document-path — шлях до параметрів, які зберігаються в даних. Шлях можна визначити за результатами виконання в Логах процесу.

        • рядок функції, що приймає два аргументи: documents (масив документів по процесу) та events (масив подій по процесу), і повертає необхідне значення параметра.

      2) accessInfo — необовʼязковий параметр, який є функцією, що приймає два аргументи documents (масив документів поточного процесу) та events (масив подій поточного процесу).

      Необхідно встановити, якщо для відповідного реєстру потрібно фіксувати дані (ПІБ, ідентифікатор користувача) по історії змін (створення, видалення, редагування записів) в Електронному кабінеті.

      Сервіс Liquio Event додатково фіксує workflowId (ідентифікатор поточного процесу) та requestAt (дата та час здійснення дії відносно реєстру) при виконанні функції.
