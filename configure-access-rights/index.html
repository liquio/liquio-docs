    Як налаштувати права доступу?

    Керування підрозділами та користувачами

    Передумови

    Права доступу користувача в загальному випадку налаштовуються на рівні підрозділу, до якого він належить.

    Підрозділ — це група користувачів із визначеними правами доступу. Користувач може належати до підрозділу або бути його керівником. Перед налаштуванням прав доступу необхідно створити підрозділ користувачів та додати потрібного користувача до нього.

    Створення підрозділу

    Порядок дій:

    1) Перейдіть у розділ Користувачі > Користувацькі підрозділи.

    2) Натисніть кнопку Створити.

    3) У спливному вікні Створення підрозділу заповніть поля:

    • Назва підрозділу;

    • ID підрозділу (необов’язково);

    • Опис підрозділу (необов’язково).

    4) Натисніть кнопку Зберегти.

    Додавання користувача в підрозділ

    Порядок дій:

    1) Оберіть один з наявних підрозділів.

    2) Натисніть кнопку Додати користувачів.

    3) Здійсніть пошук користувача за ПІБ, РНОКПП або ID.

    4) Натисніть кнопку Додати навпроти користувача.

    Права доступу до задач

    Права доступу до задачі процесу визначають, які користувачі бачитимуть цю задачу під час проходження процесу в електронному кабінеті.

    Для налаштування прав доступу:

    1) У редакторі процесу перейдіть на вкладку Права доступу.

    2) Налаштуйте JavaScript-функцію, що визначає права доступу.

    За замовчуванням створюється функція, яка визначає одну задачу, доступну для всіх користувачів, з якої буде починатись процес:

    1
2
3
    {
   () => 20010001;
}

    Також є можливість розрахувати різні початкові задачі для різних підрозділів і надати доступ до процесу тільки для визначених підрозділів:

    1
2
3
    {
   (user, unitIds) => { if (unitIds.some(item => (item === 11 || item === 17 || item === 7))) { return 2002001; }};
}

    Варіант функції розрахунку, за якої початкова задача – 3121001, але за умови, що поточний користувач є керівником підрозділу 2:

    1
2
3
    {
   (user, unitIds, units) => { if (units.head.some(v => v.id === 2)) return 3121001; }
}

    Права доступу до реєстрів

    Для налаштування доступу на рівні ключа:

    1) На вкладці Реєстри оберіть потрібний реєстр. Відбувається перехід до налаштувань ключів реєстру.

    2) Для потрібного ключа реєстру натисніть на піктограму Доступ.

    3) У спливаючому вікні під заголовком Доступ через редактор реєстрів виконайте такі налаштування:

    • Оберіть необхідний підрозділ.

    • Активуйте перемикачі навпроти дій, які користувач зможе виконати із записом:

      • Перегляд;

      • Створення;

      • Зміна;

      • Видалення;

      • Перегляд історії змін;

      • Приховати в переліку ключів в редакторі реєстрів;

    • Оберіть, хто може вчиняти дії з записами:

      • Усі члени підрозділу;

      • Тільки керівники.

    Підказка

    За необхідності доступ можна описати для декількох різних підрозділів. Для цього натисніть на кнопку Додати підрозділ.

    4) Натисніть кнопку Зберегти.

    Для активування можливості використовувати ключ реєстру в Електронному кабінеті на формах вводу в якості джерела даних встановіть перемикач Використовувати на формах вводу (strict access). Це налаштування є незалежним від підрозділу користувачів.

    Підказка

    Налаштування ключа реєстра Використовувати на формах вводу (strict access) та Містить персональні дані є несумісними, тобто їх неможливо активувати одночасно.
