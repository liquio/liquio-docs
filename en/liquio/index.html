    What is Liquio?

    Liquio is a low-code platform for the digital transformation of public services and the rapid development of software solutions for the public sector with minimal manual coding.

    Key capabilities of Liquio

    • Rapid creation and configuration of digital services — the low-code approach and ready-made modules reduce dependency on backend developers and accelerate the launch of new services.

    • Automation tools with AI support — AI components assist with creating initial process diagrams, provide guidance during development, and power a chat assistant.

    • Support for security requirements and high loads — the platform is certified to work with high-security resources and is designed for stable operation under significant load.

    • Ready-made integrations and API-first architecture — integration with government registers is supported, as well as interaction with other systems via APIs.

    • Analytics and service monitoring tools — analytical components enable tracking the status and performance of digital services.

    • Centralized access via a web portal and mobile solutions — users access digital services and documents through a unified web interface and mobile channels.

    • Open-source and vendor independence — the platform is based on open-source software and can be deployed and maintained independently.

    Liquio user-facing components

    Liquio has two primary user-facing components that support different roles: the Admin Panel and the User Workspace.

    graph LR
    A[Liquio] --> B[Admin Panel]
    A[Liquio] --> C[User Workspace]

    Admin Panel — the Liquio platform user-facing component for development teams. The Admin Panel enables building the logic of services available to users in the User Workspace.

    User Workspace — the Liquio platform user-facing component for end users of services built on the platform.

    Liquio’s modular architecture enables a seamless workflow: processes configured by low-code developers in the Admin Panel are automatically deployed as services that end users can access through the User Workspace.

    What is a process?

    Process is an ordered sequence of actions aimed at achieving a specific goal, with clearly defined responsibilities and expected outcomes. In the Liquio Admin Panel, a process is represented as a BPMN diagram that visualizes steps, decisions, parallel actions, and participant roles.
