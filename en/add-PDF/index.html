    How to Add PDF?

    Why is PDF needed?

    A task can end with document generation: an application, letter, etc. A PDF file is created based on data entered by the user using an HTML template.

    For example, when submitting an application, the user fills in the necessary data at the task steps, and at the end signs the generated PDF application document, which contains the entered data and consent to submit the application.

    Task Settings

    Parameters configured at the task level:

    1) pdfRequired — determines whether a PDF will be generated at the end of the task.

    2) signRequired — determines whether the document needs to be signed using a digital signature.

    Example of settings where a PDF is generated at the last step of the task without requiring a signature:

    1
2
3
4
    {
  "pdfRequired": true
  "signRequired": false
}

    PDF Template Settings

    The content and appearance of the PDF is configured in a separate JSON file on the Pdf document template tab.

    PDF is generated based on an HTML code block. To create HTML content inside the file, the following are used:

    • HTML and CSS: configures the general appearance of the PDF;
    • Handlebars: configures conditions for displaying fields.

    HTML code is converted to a PDF file using the playwright library.

    Connecting third-party libraries for html is not possible

    PDF Structure

    PDF consists of three main parts:

    • PDF Header — may contain logos, a title like "Application No.99999" and a subtitle with arbitrary text. This part of the PDF may also contain a QR code.

    • General Information Sections — may consist of several items that contain corresponding data fields. Most often, items in the general information section are directly the steps that the user filled out or viewed.

    • PDF Footer — by default contains the current date and the applicant's full name — place for signature. If there are two or more signatories, they are listed.
